## Gotchas for resuming

- `port/port_imgui_menu.cpp` forward-declares `Port_Audio_*` (~line 350) and the
  `Port_Config_*` binding API (~line 384) **locally**, NOT via header. Add new
  audio/binding decls there too, even though it includes `port_runtime_config.h`
  and `port_debug_actions.h` (so Config getters + debug actions DO come via header).
- Config persistence pattern (issue #146): member (`sXxx`) + `DefaultsJson` entry
  + load (`j.value`) + getter/setter (`extern "C"`, setter calls `SaveConfig()`) +
  header decl. Mirror `Port_Config_GetVSync` / `sVSyncCfg`.
- Build/verify both configs and restore: `xmake f --gpu_renderer=y && xmake build
  -y tmc_pc`, then `xmake f --gpu_renderer=n` to restore. Current default is
  `gpu_renderer=false`, `multi_region=true`, `widescreen_width=240`.
- GPU-only code must sit behind `#ifdef TMC_GPU_RENDERER` and be verified in the
  GPU config (the software build won't compile it). `Port_GPU_PresentPrelaunchFrame`
  renders an ImGui frame on the SDL_GPU swapchain.
- Debug-action coord write: `gPlayerEntity.base.x/y.HALF.HI = px` (world pixels);
  gate on `gSave.stats.health != 0 && gPlayerState.framestate != PL_STATE_DIE`.
